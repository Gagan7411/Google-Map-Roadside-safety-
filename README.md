## ðŸ“¡ Feature 2: Off-Grid SOS Protocol (Technical Failsafe)

This feature is the core technical innovation, guaranteeing communication when a standard voice call is impossible.

| Component | Technical Action | Impact on Safety |
| :--- | :--- | :--- |
| **Off-Grid SOS Activation** | Activated when the device detects a total cellular dead zone. The user selects the emergency type (Medical/Breakdown/Accident). | **Guaranteed Lifeline:** Bypasses the single point of failure in the existing 1033 voice-based system. |
| **Low-Bandwidth Data Burst** | The app attempts to send a highly compressed data packet ($< 50$ bytes) containing only **GPS Coordinates** and the **Emergency Type**. | **Resilience:** A minimal data packet has a significantly higher success rate in pushing through fringe signals than a high-bandwidth voice call. |
| **Direct NHAI API Routing** | The signal is routed through a proposed private API connection between Google and the NHAI Incident Management Control Centre. | **Fastest Dispatch:** Ensures location and issue data are received immediately, drastically reducing the time-to-dispatch. |
